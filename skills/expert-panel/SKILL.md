# Expert Panel

분야별 전문가 에이전트를 병렬로 실행하여 최신 트렌드 기반 다각도 리뷰를 수행하는 시스템.

## 사용법

```
/expert-panel "결제 플로우가 복잡한 것 같아"              # 자동 추천
/expert-panel --experts "security,ux,fe-arch" "인증 검토"  # 직접 지정
/expert-panel --panels "frontend,business" "랜딩페이지"     # 패널 단위
/expert-panel --all "서비스 런칭 전 종합 검토"              # 전체 투입
```

## 실행 프로세스

### Step 1: 입력 파싱

`$ARGUMENTS`에서 플래그와 주제를 분리한다.

**플래그 처리:**
- `--experts "a,b,c"` → 전문가 직접 지정 모드
- `--panels "x,y"` → 패널 단위 지정 모드
- `--all` → 전체 15명 투입 모드
- 플래그 없음 → 자동 추천 모드

**전문가 ID 매칭:**

`references/panels.md`의 별칭 매핑 테이블을 참조하여 입력값을 에이전트 ID로 변환한다.

1. 입력값이 `expert-`로 시작하면 그대로 사용
2. `expert-` 접두사 없으면 자동 추가하여 매칭 시도 (예: `security` → `expert-security`)
3. 영문 별칭 매칭 (예: `fe-arch` → `expert-frontend-architect`)
4. 한글 별칭 매칭 (예: `보안` → `expert-security`)

### Step 2: 전문가 선정

**자동 추천 모드 (플래그 없음):**

주제를 분석하여 적합한 전문가 3-5명을 선정한다. 다음 형식으로 사용자에게 제시:

```
주제를 분석한 결과, 다음 전문가를 투입하겠습니다:

| 전문가 | 선정 이유 |
|--------|---------|
| @expert-ui-ux-designer | 사용자 플로우 관련 UX 검토 필요 |
| @expert-frontend-architect | 프론트엔드 구조 검증 필요 |
| @expert-strategy | 비즈니스 관점 검토 필요 |

진행할까요? (전문가를 추가/제거하려면 알려주세요)
```

**사용자 확인을 받은 후에만 다음 단계로 진행한다.**

**직접 지정 모드 (`--experts`):**
별칭 매핑 후 바로 Step 3으로 진행.

**패널 단위 모드 (`--panels`):**
`references/panels.md`에서 해당 패널의 전문가 목록을 가져와 바로 Step 3으로 진행.

**전체 투입 모드 (`--all`):**
먼저 경고 메시지를 출력:
```
15명 전체 투입은 시간이 걸릴 수 있습니다. 진행할까요?
```
사용자 확인 후 Step 3으로 진행.

### Step 3: 병렬 에이전트 디스패치

확정된 전문가 목록에 따라 에이전트를 병렬로 실행한다.

**배치 전략:**

| 투입 인원 | 실행 방식 |
|----------|---------|
| 1-5명 | 전원 동시 병렬 실행 |
| 6-10명 | 5명씩 2배치 순차 실행 |
| 11-15명 | 5명씩 3배치 순차 실행 |

**디스패치 방법:**

각 전문가를 Agent 도구로 호출한다. 모든 에이전트에게 동일한 주제와 검토 대상을 전달:

```
Agent 도구 호출:
  subagent_type: {에이전트 ID}  (예: expert-security)
  prompt: |
    검토 주제: {사용자가 입력한 주제}
    검토 대상: {관련 파일 경로 또는 설명}

    위 주제에 대해 당신의 전문 분야 관점에서 검토해주세요.
    Phase 1-4 프로세스를 따라 실행하고, 출력 형식에 맞춰 리포트를 작성해주세요.
```

**같은 배치 내의 에이전트는 반드시 하나의 메시지에서 병렬로 호출한다.** 배치가 여러 개인 경우, 이전 배치의 모든 에이전트가 완료된 후 다음 배치를 실행한다.

### Step 4: 결과 통합

모든 에이전트의 리포트를 수집하여 통합 리포트를 생성한다.

**통합 리포트 형식:**

```markdown
# Expert Panel Report
> 주제: "{사용자 입력 주제}"
> 투입 전문가: {전문가 목록}
> 날짜: {실행일}

---

## 종합 요약
| 등급 | 개수 |
|------|------|
| 🔴 Critical | N |
| 🟡 Warning | N |
| 🟢 Good | N |
| 💡 Trend Insight | N |

## 🔴 Critical (즉시 조치)
1. [{전문가}] {내용}

## 🟡 Warning (개선 권장)
1. [{전문가}] {내용}

## 💡 Trend Insights (최신 트렌드 기반)
1. [{전문가}] {트렌드 인사이트} (출처: {URL})

## 🟢 Good (잘 된 점)
1. [{전문가}] {내용}

## 액션 아이템 (우선순위순)
1. [ ] {실행 가능한 개선 사항}
```

**통합 시 규칙:**
- 여러 전문가가 동일한 이슈를 지적한 경우 하나로 합치고 관련 전문가를 모두 태그
- Critical > Warning > Trend Insight > Good 순서로 정렬
- 액션 아이템은 Critical과 Warning에서 추출, 구체적이고 실행 가능한 형태로 작성
- 에이전트가 응답하지 않은 경우 "⏱ {전문가}: 응답 없음" 으로 표기

## 5개 패널 구성

`references/panels.md` 참조.

| 패널 | 전문가 |
|------|--------|
| **ui** | UX 디자이너, 비주얼 디자이너, IA 전문가 |
| **frontend** | 프론트엔드 아키텍트, 성능 전문가, 접근성 전문가 |
| **backend** | 백엔드 아키텍트, 보안 전문가, DB 성능 전문가 |
| **infra** | DevOps, 클라우드 아키텍트, SRE |
| **business** | 전략 전문가, 마케팅 전문가, PM/기획자 |

## 기존 시스템과의 관계

| 상황 | 사용할 도구 |
|------|-----------|
| PR 전 코드 품질/보안/버그 빠른 체크 | `/agent-teams review` |
| 최신 트렌드 기반 다각도 심층 리뷰 | `/expert-panel` |
| 개별 전문가 의견이 필요할 때 | `@expert-{id}` 직접 호출 |
